Apart from unit tests and stress tests, you can add fuzzers to [this directory](https://github.com/facebook/rocksdb/blob/main/fuzz) by leveraging the fuzzing infrastructure there, the fuzzers are continuously run on Google's [OSS-Fuzz platform](https://github.com/google/oss-fuzz/tree/master/projects/rocksdb), errors can be found at [Chromium's oss-fuzz website](https://bugs.chromium.org/p/oss-fuzz/issues/list?sort=-opened&q=proj%3Arocksdb&can=1). and will be reported to RocksDB developers through emails. See more details in [this README](https://github.com/facebook/rocksdb/blob/main/fuzz/README.md).
