Added experimental ExternalLogFileManager APIs for RocksDB-managed, MANIFEST-tracked external log files. The application-provided external log file name is both the public lookup key and physical path persisted in MANIFEST; kDBRelativePath resolves it under the DB directory, while kExternalPath treats it as an explicit external path. The API supports bulk ingest and bulk delete with one MANIFEST edit per call. Existing DBs reopened with this version are unaffected unless an external log file is created or ingested. Once a DB uses the new external-log MANIFEST records, older RocksDB versions cannot reopen that MANIFEST.
