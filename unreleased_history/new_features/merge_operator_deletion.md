`MergeOperator::FullMergeV3()` can now produce a deletion result by setting `MergeOperationOutputV3::new_value` to `std::monostate`. When the merge operator chooses deletion, the key behaves as if it were tombstoned: `Get()` / `MultiGet()` / `Transaction::Get()` / `WriteBatchWithIndex::GetFromBatchAndDB()` return `Status::NotFound()` (with subcode `kMergeOperatorDeletion` to distinguish from a real "key absent"), forward and reverse iterators skip the key, and compaction either emits a `kTypeDeletion` tombstone or drops the key entirely at the bottommost level when the full history has been seen. This enables counter auto-delete-at-zero, conditional mutations, and Cassandra-style row expiration directly from a merge operator, without requiring a separate compaction filter. On-disk SST format is unchanged; the deletion is materialized as an ordinary tombstone so SSTs remain readable by older RocksDB binaries.
