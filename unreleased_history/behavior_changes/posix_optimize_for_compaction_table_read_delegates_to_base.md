`PosixFileSystem::OptimizeForCompactionTableRead` now delegates to the base `FileSystem::OptimizeForCompactionTableRead` implementation before applying its Linux-only compaction-readahead clamp (added for #12038). The returned `FileOptions::use_direct_reads` therefore reflects `DBOptions::use_direct_reads`, consistent with the base class. On Linux this means the `compaction_readahead_size` clamp is now skipped when direct reads are enabled (direct I/O bypasses kernel readahead). Previously the override skipped the base implementation and keyed both the returned options and the readahead clamp off the incoming `FileOptions` instead of `DBOptions`.
