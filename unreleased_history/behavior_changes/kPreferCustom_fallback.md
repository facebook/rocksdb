`ReadOptions::read_index = kPreferCustom` (formerly the unnamed "use UDI" override in the pre-refactor API) is now an explicit best-effort selector rather than a strict one. For SSTs that have a UDI block on disk the read is strict -- iterator-creation failures surface as `Status::Corruption` -- but SSTs that lack a UDI block (e.g., legacy files still on disk during a migration to a UDI-enabled table factory) transparently fall through to the standard index instead of erroring out. This is the behavior production deployments were already relying on; the rename to `kPreferCustom` documents it.
