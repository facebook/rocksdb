`FileSystem::OptimizeForCompactionTableRead` (and the legacy `Env::OptimizeForCompactionTableRead`) now set `FileOptions::use_direct_reads` when either `DBOptions::use_direct_reads` or the new `DBOptions::use_direct_io_for_compaction_reads` is true. Custom `FileSystem`/`Env` implementations that override these methods must set `use_direct_reads = true` in the returned options whenever `use_direct_io_for_compaction_reads` is true, otherwise the new option becomes a silent no-op for that FileSystem. The hook is also invoked by `BackupEngine` when copying SST files, so backup reads will also route through O_DIRECT when the new flag is enabled.
