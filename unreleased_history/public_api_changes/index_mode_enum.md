Replaced the experimental UDI booleans `BlockBasedTableOptions::use_udi_as_primary_index`, `skip_standard_index`, and `fail_if_no_udi_on_open` with a single `BlockBasedTableOptions::index_mode` enum (`kStandardOnly`, `kStandardDefault`, `kCustomDefault`, `kCustomOnly`). The removed bool fields are still accepted as OPTIONS-file aliases and translate monotonically to `index_mode` at parse time, so existing OPTIONS files written by older binaries continue to produce the same SST shape and read routing after upgrade (`use_udi_as_primary_index=true` -> `kCustomDefault`; `skip_standard_index=true` -> `kCustomOnly`; `fail_if_no_udi_on_open=true` -> at least `kStandardDefault`). Code that referenced the removed fields directly will not compile and must migrate to `index_mode`.
