Refactored the experimental User-Defined Index (UDI) public surface into a unified pluggable `IndexFactory` SPI (`include/rocksdb/index_factory.h`). The previously experimental class names (`UserDefinedIndexFactory`, `UserDefinedIndexBuilder`, `UserDefinedIndexReader`, `UserDefinedIndexIterator`, `UserDefinedIndexOption`) are kept as type aliases in `include/rocksdb/user_defined_index.h` so existing `shared_ptr<UserDefinedIndexFactory>` users compile unchanged. Custom subclasses that override the old `NewBuilder()` / `NewReader(Slice&)` virtuals must migrate to the option-taking signatures (`NewBuilder(const IndexFactoryOptions&, std::unique_ptr<IndexFactoryBuilder>&)` and the matching `NewReader`); the pure virtuals are now those new ones. Custom builders serialize one self-contained index block through `Finish()`, matching the single block passed to `NewReader()`. Code that references UDI value categories must also qualify the now-scoped enum values, for example `IndexFactoryBuilder::ValueType::kValue` instead of `UserDefinedIndexBuilder::kValue`. The on-disk meta block prefix is unchanged (`rocksdb.user_defined_index.`) so all existing UDI SSTs remain readable.
