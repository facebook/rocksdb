Refactored the experimental User-Defined Index (UDI) public surface into a unified pluggable `IndexFactory` SPI (`include/rocksdb/index_factory.h`). The previously experimental class names (`UserDefinedIndexFactory`, `UserDefinedIndexBuilder`, `UserDefinedIndexReader`, `UserDefinedIndexIterator`, `UserDefinedIndexOption`) are kept as type aliases in `include/rocksdb/user_defined_index.h` so existing `shared_ptr<UserDefinedIndexFactory>` users compile unchanged. Custom subclasses that override the old `NewBuilder()` / `NewReader(Slice&)` virtuals continue to compile through compatibility adapters. New implementations can override the option-taking signatures (`NewBuilder(const IndexFactoryOptions&, std::unique_ptr<IndexFactoryBuilder>&)` and the matching `NewReader`) when they need `IndexFactoryOptions` or `Status` error returns. The old value category constants, such as `UserDefinedIndexBuilder::kValue`, remain available and map to the scoped `IndexFactoryBuilder::ValueType` values. The on-disk meta block prefix is unchanged (`rocksdb.user_defined_index.`) so existing UDI SSTs remain readable.
