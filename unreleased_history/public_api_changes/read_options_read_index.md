Added `ReadOptions::read_index` (an enum `kDefault` / `kBuiltin` / `kPreferCustom`) as the preferred per-read selector between the standard index and the custom `IndexFactory` index, layered on top of `BlockBasedTableOptions::index_mode`'s default routing. Existing code that set `read_options.table_index_factory = trie_factory.get()` should migrate to `read_options.read_index = ReadOptions::ReadIndex::kPreferCustom` when it does not need to enforce a specific factory name. The legacy `table_index_factory` pointer remains as a compatibility guard: when set, RocksDB verifies the requested factory name matches the SST's custom index name and returns `InvalidArgument` instead of silently using a different custom index. `kPreferCustom` falls back to the standard index for SSTs that don't have a UDI block (e.g., older files still on disk during migration).
