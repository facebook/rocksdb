Replaced `ReadOptions::table_index_factory` (raw pointer to a `UserDefinedIndexFactory`) with `ReadOptions::read_index` (an enum `kDefault` / `kBuiltin` / `kPreferCustom`). The enum value selects between the standard index and the custom `IndexFactory` index per read, layered on top of `BlockBasedTableOptions::index_mode`'s default routing. Existing code that set `read_options.table_index_factory = trie_factory.get()` should set `read_options.read_index = ReadOptions::ReadIndex::kPreferCustom` instead (the factory itself is now configured table-wide via `BlockBasedTableOptions::user_defined_index_factory`). `kPreferCustom` falls back to the standard index for SSTs that don't have a UDI block (e.g., older files still on disk during migration).
